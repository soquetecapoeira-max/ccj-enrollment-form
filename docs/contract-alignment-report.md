## 2. 紙とWebで不一致だった項目一覧

| 項目 | 紙（旧規約） | Web（現行） | 判定 |
|---|---|---|---|
| 年会費徴収月 | 毎年4月 | 毎年3月 | **運用方針として3月に統一** |
| 支払方法の主軸 | 口座振替中心（NSS） | 口座振替（NSS）主軸 + PayPal / 現金(ドロップイン) | **整合（運用更新反映）** |
| 退会手続き | 電話/メールのみ不可、予約必須 | 要約記載 + 退会手続き事前連絡フォーム追加 | **整合** |
| プラン変更期限 | 1.5か月前 | 要約記載あり（今回追記） | 概ね整合 |
| RODA別料金 | 明記あり | 「その他イベントは別料金」に表現統一 | **整合（文言変更）** |
| 家族2人目以降 | 入会金無料（紙に記載） | 「同一家族2人目以降は入会金無料」を明記 | **整合** |

> 注記: 年会費徴収月は運用方針として3月に統一。紙規約の旧記載（4月）は運用版規約への改定で吸収します。

## 3. 今後API連携するための次ステップ

1. `formType: "enrollment"` + `payload` 形式へ統一（既存フラットJSONと並行受け）
2. 料金・規約文言を1ソース化（JSON/定数ファイル）し、画面・メールで共通利用
3. `termsVersion` を必須にして、送信時に規約バージョンを固定記録
4. 完了画面の支払方法別案内を、将来の決済URL/口座登録URLに差し替え可能な設定化
5. 自動返信メールテンプレを `paymentMethod` で分岐可能にする
6. バックエンド（GAS）で入力正規化・形式検証を二重化（フロント依存を避ける）
